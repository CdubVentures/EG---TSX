# Folder Map

> **Purpose:** Annotated directory tree so an LLM knows where everything lives without scanning the filesystem.
> **Prerequisites:** [Scope](scope.md)
> **Last validated:** 2026-03-18

## Root

```text
EG - TSX/
├── src/                          # Application source code (see src/README.md)
├── public/                       # Static assets (fonts, images, JS) — served by Astro
├── config/                       # Python config tool + mutable JSON config (see config/README.md)
├── docs/                         # Architecture & reference documentation
├── scripts/                      # Build, deploy, migration, and validation scripts (see scripts/README.md)
├── test/                         # Test suite (Node.js built-in runner)
├── infrastructure/               # AWS IaC and checked-in deploy artifacts (see infrastructure/README.md)
├── tools/                        # Local utility apps, including deploy-dashboard
├── dist/                         # Build output (generated by `npm run build`)
│
├── astro.config.mjs              # Astro framework configuration
├── tsconfig.json                 # TypeScript config (strict, path aliases)
├── package.json                  # npm dependencies and scripts
├── lambda-entry.mjs              # AWS Lambda entry point (proxies to Astro standalone)
├── .env / .env.example           # Environment variables (secrets / template)
└── README.md                     # Project entry point
```

Repo-level agent rules live outside the app at [`../AGENTS.md`](../../../AGENTS.md).

## src/ — Application Source

```text
src/
├── core/                         # Shared kernel — data gateways, config, image resolution
│   ├── config.ts                 # Centralized behavioral knobs (pagination, CDN, categories)
│   ├── categories.ts             # Category color derivation engine (HSL math from JSON)
│   ├── category-contract.ts      # Category validation boundary (Zod)
│   ├── products.ts               # Product data gateway (filters by active categories)
│   ├── content.ts                # Article collection queries (reviews, guides, news, etc.)
│   ├── dashboard.ts              # Editorial dashboard config gateway
│   ├── images.ts                 # Universal image URL resolver (contentImage, collectionImagePath)
│   ├── media.ts                  # Product media helpers (getImage, getCarouselImages, getImageForColor)
│   ├── hub-tools.ts              # Hub tools data gateway (respects category flags)
│   ├── article-helpers.ts        # URL construction, hero resolution, date formatting
│   ├── product-helpers.ts        # Product utility functions
│   ├── cache-cdn-contract.ts     # CDN cache header contract
│   ├── db.ts                     # PostgreSQL connection (search DB)
│   ├── seo/                      # SEO: route graph, sitemap, robots, indexation policy
│   ├── routing/                  # URL slug utilities
│   └── tests/                    # Colocated core module tests
│
├── features/                     # Feature boundaries — NO cross-feature imports
│   ├── auth/                     # Cognito OAuth, JWT, session (SHARED KERNEL for user state)
│   ├── home/                     # Home page: slideshow, dashboard, featured scroller, news
│   ├── vault/                    # User saved items: DynamoDB sync, merge, thumbnails
│   ├── search/                   # Full-text search: PostgreSQL, SearchDialog
│   ├── settings/                 # Theme selection, user preferences (localStorage)
│   ├── notifications/            # Toast queue, vault bridge
│   ├── site-index/               # Category/brand index pages, pagination, structured data
│   └── ads/                      # Ad placement, inline ads (rehype plugin), sample ads
│
├── shared/                       # Reusable UI & utilities — features may import, not vice versa
│   ├── layouts/                  # MainLayout, GlobalNav, NavLinks, NavIcons, NavMobile, SiteFooter
│   ├── ui/                       # AnnouncementBar, Breadcrumbs, PinTag, Pagination, badges
│   └── lib/                      # cn.ts (clsx + tailwind-merge)
│
├── pages/                        # Astro file-based routing
│   ├── index.astro               # Home page
│   ├── 404.astro                 # Not found
│   ├── robots.txt.ts             # Dynamic robots.txt
│   ├── api/                      # SSR API endpoints (search, auth, vault, admin)
│   ├── brands/[...slug].astro    # Brand index pages
│   ├── guides/[...slug].astro    # Guide articles
│   ├── news/[...slug].astro      # News articles
│   ├── reviews/[...slug].astro   # Product reviews
│   ├── auth/callback.ts          # OAuth callback
│   ├── login/                    # OAuth login routes (Google, Discord)
│   └── logout.ts                 # Logout handler
│
├── content/                      # Astro content collections
│   ├── data-products/            # 366 product JSON files (mouse/keyboard/monitor)
│   ├── reviews/                  # 47 review articles (slug-folder MDX)
│   ├── guides/                   # 33 guide articles
│   ├── news/                     # 23 news articles
│   ├── games/                    # 11 game articles
│   ├── brands/                   # 29 brand profiles
│   └── pages/                    # 4 static pages
│
├── styles/                       # Global CSS (NOT per-component)
│   ├── global.css                # Theme tokens, CSS variables, resets, base styles
│   ├── dialogs.css               # Popover/dialog styling
│   └── nav-mobile.css            # Mobile nav styles
│
├── types/                        # TypeScript declarations
│   ├── vendor.d.ts               # Third-party globals (VanillaTilt, googletag)
│   └── test-overrides.d.ts       # Test-specific type overrides
│
├── middleware.ts                  # Astro request middleware (JWT validation, token refresh)
├── content.config.ts             # Content collection Zod schemas
└── env.d.ts                      # Astro environment types
```

## scripts/ — Build & Migration

```text
scripts/
├── run-astro-build.mjs           # Production build wrapper with validation
├── deploy-aws.mjs                # CloudFormation deploy + Lambda upload
├── build-media.mjs               # Product media schema builder (re-runnable)
├── sync-db.mjs                   # PostgreSQL search index sync
├── sync-rename.mjs               # Content + image folder rename (atomic)

└── archive/                      # One-time migration scripts (preserved)
```

## test/ — Test Suite

~50 test files using Node.js built-in `node --test` runner. Tests are `.test.mjs` files. Additional ~20+ colocated tests in `src/features/*/tests/` and `src/core/tests/`.
